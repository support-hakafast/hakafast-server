## 2. ציוד ודרישות לפני הגעה

### חומרה חובה

- [ ] מחשב Windows 10/11 (64-bit) — **מחשב האחראי** (מומלץ i5+, 8GB RAM, SSD)
- [ ] כבל רשת (Ethernet) מהמחשב לרשת המסלול

- שירות Windows **HAKAFAST** — עולה עם המחשב
- WebView2 Kiosk (אופציונלי) — מסך מלא להזמנות

## 4. שלב 1 — סקר שטח ותשתית

**מטרה:** למפות פיזית את המסלול לפני חיבור תוכנה.

### 4.1 רשת

1. סמן את מיקום **מחשב האחראי** (קרוב לדקודר ולחשמל יציב)
2. בדוק שיש נקודת Ethernet / מתג
3. תכנן IP קבוע למחשב האחראי, למשל: `192.168.1.100`
4. וודא ש-Wi-Fi מסלול מאפשר גישה בין מכשירים (AP isolation **כבוי**)

### 4.2 חשמל

- [ ] UPS למחשב + דקודר
- [ ] שקעים ליד מסכים

### 4.3 פיטס ומסלול

1. ספור נתיבי יציאה (lanes) — כמה שורות פיטס
2. סמן כיוון יציאה (למעלה/למטה במסך) — יוגדר בהגדרות מתקדמות
3. צלם תמונות: דקודר, לולאות, מסכים, ארון תקשורת

### 4.4 דקודר

1. מצא את הדקודר — רשום MAC / Serial אם רלוונטי
2. חבר Ethernet לדקודר
3. בדוק IP (מסך דקודר / MyLaps software / סריקה):
   ```powershell
   ping 192.168.1.50
   ```
4. פורט ברירת מחדל: **5403** (TCP)

**✅ סימון שלב:** תרשים רשת + IP לכל רכיב + תמונות

## 5. שלב 2 — התקנת תוכנה

**מטרה:** שרת HAKAFAST רץ על מחשב האחראי.

### 5.1 הרצת המתקין

1. חבר USB, הרץ `HAKAFAST-Setup-x.x.x.exe`
2. אשר UAC (נדרש Admin)
3. בחר:
   - [x] **Install Windows service**
   - [x] **Create desktop shortcuts**
4. מיקום: `C:\Program Files\HAKAFAST`

### 5.2 אשף הגדרה ראשונית

לאחר ההתקנה ייפתח דפדפן ב:

```
http://127.0.0.1:5000/setup
```

מלא:

| שדה | דוגמה |
|-----|--------|
| שם מסלול | Karting Tel Aviv |
| Slug | `kart-ta` |
| מספרי קארט | `1-12` |
| סיסמת מנהל | (חובה מומלץ — 12+ תווים, אות גדולה/קטנה, מספר, תו מיוחד) |

לחץ **שמור** — שמור את כתובות ה-LAN שמוצגות.

### 5.3 בדיקה מהירה

```powershell
Get-Service HAKAFAST
# Status: Running

Start-Process "http://127.0.0.1:5000/admin/kart-ta"
```

### 5.4 IP קבוע (מומלץ)

1. הגדרות Windows → רשת → Ethernet → IP assignment → Manual
2. IP: `192.168.1.100` · Subnet: `255.255.255.0` · Gateway: `192.168.1.1`

**✅ סימון שלב:** שירות Running + אשף הושלם + כניסה לפאנל ניהול

## 6. שלב 3 — חיבור דקודר ו-TranX 160

**מטרה:** HAKAFAST מקבל הקפות מהדקודר.

> HAKAFAST **לא** מתחבר ישירות ללולאה — הלולאה → טרנספונדר → דקודר → TCP → HAKAFAST.

### 6.1 חיבור פיזי TranX 160

1. לולאות מחוברות לדקודר לפי מדריך AMB/MyLaps
2. דקודר מחובר לרשת (Ethernet)
3. הדקודר והמחשב **באותה רשת מקומית**

### 6.2 הגדרה בפאנל ניהול

1. פתח **פאנל ניהול** → **הגדרות מתקדמות** (סיסמה אם הוגדרה)
2. גלול ל-**הגדרות דקודר**
3. לחץ **סרוק רשת** — בחר את הדקודר מהרשימה  
   **או** הזן ידנית:
   - **IP דקودר:** `192.168.1.50`
   - **פורט:** `5403`
   - **COM** (USB serial): רק אם אין Ethernet — למשל `COM3`
4. לחץ **שמור**
5. וודא נקודה ירוקה: **מחובר**

### 6.3 בדיקת חיבור

1. העבר קארט עם טרנספונדר מעל לולאה
2. בדוק **זמן מעבר אחרון** מתעדכן בהגדרות דקודר
3. אם לא — ראה [פתרון תקלות](#14-פתרון-תקלות)

### 6.4 חומת אש (Firewall)

```powershell
New-NetFirewallRule -DisplayName "HAKAFAST Server" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
New-NetFirewallRule -DisplayName "AMB Decoder" -Direction Inbound -Protocol TCP -LocalPort 5403 -Action Allow
```

**✅ סימון שלב:** סטטוס דקודר = מחובר + מעבר טרנספונדר נרשם

## 7. שלב 4 — מיפוי טרנספונדרים וקארטים

**מטרה:** כל ID טרנספונדר = מספר קארט נכון.

### 7.1 איסוף ID טרנספונדרים

לכל קארט (1 עד N):

1. הזז קארט מעל לולאה **ב slowly**
2. רשום את **Transponder ID** (מופיע ב-MyLaps / בלוג דקודר)
3. מלא בטבלת המיפוי:

| טרנספונדר | קארט |
|-----------|------|
| 1234567 | 1 |
| 2345678 | 2 |
| … | … |

### 7.2 הזנה ב-HAKAFAST

**הגדרות מתקדמות → מיפוי טרנספונדר → קארט**  
שמור.

### 7.3 הגדרת קארטים בפאנל

1. פאנל ניהול → מחסן קארטים
2. וודא שכל מספר קארט קיים
3. (אופציונלי) הגדר סוגי קארט — צבע, מנוע, טווח מספרים

### 7.4 מקצה בדיקה

1. הוסף 2–3 נהגים לתור
2. שבץ לקארטים
3. התחל מקצה
4. וודא שהקפות מופיעות בזמנים חיים

**✅ סימון שלב:** כל קארט נקרא + הקפה מוצגת תוך ~2 שניות

## 14. פתרון תקלות

### השרת לא עולה

```powershell
Get-Service HAKAFAST
Start-Service HAKAFAST
Get-Content "$env:ProgramData\HAKAFAST\logs\service.log" -Tail 30
```

או: הרץ `start-hakafast.bat admin`

### אין הקפות

1. IP דקודר נכון? → הגדרות מתקדמות
2. `ping [IP-דקודר]` מהמחשב
3. טרנספונדר על הקארט?
4. מיפוי טרנספונדר נכון?
5. Firewall — פורט 5403

### מסך לא מתעדכן

1. אותה רשת Wi-Fi?
2. AP isolation כבוי?
3. רענון F5
4. בדוק WebSocket: badge "Realtime" בכותרת

### מודal / theme לא נכון בפאנל

- כפתור 🌙/☀️ בכותרת הפאנל — כל החלונות (הגדרות, תכנון, תצוגה מקדימה) עוקבים

### הסרת התקנה

1. תפריט התחל → HAKAFAST → Uninstall  
   **או** Control Panel → Programs  
2. (אופציונלי) שמור `%ProgramData%\HAKAFAST` לפני מחיקה

## 16. אבטחה — חובה במסלול

### עקרונות

HAKAFAST במסלול רץ **ברשת מקומית (LAN)**. האבטחה מבוססת על:

1. **סיסמת מנהל חזקה** — חובה בהתקנה (12+ תווים)
2. **נעילת פאנל** — כל פעולת ניהול דורשת פתיחה עם סיסמה
3. **רשת מבודדת** — Wi-Fi צוות/VLAN נפרד מאינטרנט אורחים
4. **חומת אש Windows** — פורט 5000 רק ברשת המקומית
5. **גיבוי** — `%ProgramData%\HAKAFAST` מגובה מחוץ למחשב

### מה המערכת מגינה

| איום | הגנה |
|------|------|
| גישה לא מורשית לפאנל | סיסמה + session מוצפן |
| סיסמאות בקובץ | hashing (scrypt) — לא plaintext |
| זיוף הקפות מרחוק | API טרנספונדר — localhost בלבד |
| brute force | rate limit על login |
| רישיון מזויף | אימות מפתח (לא bypass ב-production) |
| session hijack | cookie httpOnly + secret ייחודי לכל התקנה |

### הגדרות מומלצות

```powershell
# אופציונלי — secret ייחודי (נוצר אוטומטית ב-install.json)
[Environment]::SetEnvironmentVariable('SESSION_SECRET', 'your-long-random-secret', 'Machine')

# רק אם צריך דקודר חיצוני (לא מומלץ)
# [Environment]::SetEnvironmentVariable('HF_ALLOW_REMOTE_TRANSPONDER', '1', 'Machine')
```

### אין לעשות

- ❌ לחשוף פורט 5000 לאינטרנט (port forwarding)
- ❌ להשאיר התקנה ללא סיסמת מנהל
- ❌ לשתף Wi-Fi צוות עם אורחים (AP isolation)
- ❌ לשמור סיסמאות בקובץ Excel לא מוצפן
