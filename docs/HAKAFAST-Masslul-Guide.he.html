1. מה זה HAKAFAST?

HAKAFAST הוא מערכת ניהול מקצים וזמנים חיים למסלולי קארטינג. היא מנהלת את מה שקורה במסלול: תור נהגים, שיבוץ לקארטים, יציאה מפיטס, הקפות, תוצאות, מעבר בין מקצים — ולא את מערכת הקופה או האתר.

המערכת כוללת שלושה מסכים עיקריים:

  • פאנל ניהול (Admin) — שליטה מלאה: תור, שיבוץ, פיטס, הגדרות מקצה, ייצוא תוצאות.
  • זמנים חיים (Live Timing) — תצוגה למסך גדול / QR / טלוויזיה במסלול.
  • קבלה (Reception) — מסך פשוט לרישום נהגים לתור (טאבלט בדלפק).

2. ארכיטקטורה — מי עושה מה

2.1 מנוע המקצים (Live Race Engine)

כל המידע החי — תור, מקצה נוכחי, קארטים בפיטס, הקפות — נשמר ב-workspace מקומי (קובץ JSON תחת %ProgramData%\HAKAFAST\workspaces\). זה המנוע שמנהל את היום-יום במסלול.

2.2 TranX / טרנספונדרים

חומרת הזמנים (AMB TranX, לולאות מגנטיות, טרנספונדרים בקארטים) שולחת אירועים ישירות ל-HAKAFAST: יציאה מפיטס, כניסה לפיטס, חציית קו סיום (הקפה). Rentix לא מנהל זמנים — TranX → HAKAFAST.

2.3 PostgreSQL (ענן / גיבוי)

בסיס הנתונים בענן משמש בעיקר לנתיב legacy ולגיבוי פרופיל מסלול (track_profiles). הוא אינו מנוע המקצים החי. המקצה רץ מקומית על המחשב במסלול (KIOSK).

2.4 Rentix (אופציונלי)

Rentix הוא מערכת CMS חיצונית להזמנות ותשלומים באינטרנט. HAKAFAST מתחבר אליה רק אם המסלול כבר מוכר סessions דרך Rentix ורוצה שהזמנות שולמו ייכנסו אוטומטית לתור.

3. תור נהגים — המושג המרכזי

תור נהגים (driverQueue) הוא רשימה פשוטה של נהגים שממתינים למקצה הבא. כל נהג בתור מכיל בדרך כלל:

  • שם
  • טלפון (אופציונלי)
  • רמה (Amateur / Master / Pro)
  • שם קבוצה (במקצי סיבולת — אופציונלי)
  • מקור הרישום (ידני, reception, rentix)

התור הוא אותו תור בכל המערכת — לא משנה אם הנהג נרשם בקבלה, בפאנל ניהול, או הגיע מ-Rentix. מנהל המסלול לוחץ "שבץ קארטים" והמערכת לוקחת נהגים מהתור ומשבצת אותם לקארטים פנויים.

4. קבלה (Reception) — מה זה ולמי

4.1 מה זה?

מסך Reception הוא ממשק קל ונקי לרישום נהגים שמגיעים למסלול. הוא מיועד לטאבלט או מחשב בדלפק הקבלה — לא למנהל המקצה.

כתובת (דוגמה): http://127.0.0.1:5000/reception/{track-slug}

4.2 מה אפשר לעשות בקבלה?

  • לראות כמה נהגים בתור
  • לראות מספר המקצה הנוכחי ואם יש מקצה פעיל
  • להוסיף נהג: שם, טלפון, רמה
  • להסיר נהג מהתור (טעות / ביטול)

4.3 מה אי אפשר לעשות בקבלה?

  • לשבץ קארטים
  • להתחיל / לעצור מקצה
  • לנהל פיטס
  • לקבל תשלום
  • לנהל הזמנות מהאינטרנט
  • לראות זמנים חיים מפורטים
בקיצור: Reception = "רישום שם לתור". זהו. כל השאר קורה ב-Admin.

4.4 API של הקבלה

5. תור נהגים בפאנל הניהול (Admin)

בפאנל הניהול יש את אותו תור — עם יכולות נוספות:

  • הוספה / עריכה / הסרה של נהגים
  • שיבוץ אוטומטי — "שבץ קארטים" לוקח נהגים מהתור ומשייך לקארטים
  • ניהול פיטס, יציאה, הקפות, סיום מקצה
  • מעבר למקצה הבא
  • ייצוא CSV/PDF בסיום

Reception ו-Admin משתפים את אותו מערך driverQueue בזמן אמת. נהג שנרשם בקבלה מופיע מיד בפאנל הניהול.

6. Rentix — מה זה ולמה בכלל צריך

6.1 מה זה Rentix?

Rentix (cms.rentix.biz) הוא מערכת חיצונית שמסלולים רבים בישראל משתמשים בה ל:

  • אתר הזמנות / חנות אונליין
  • תשלום מראש (כרטיס אשראי, PayPal וכו')
  • ניהול מלאי סessions ומחירים
  • CRM בסיסי ללקוחות

6.2 מה HAKAFAST עושה עם Rentix?

HAKAFAST לא מחליף את Rentix. הוא רק מקבל ממנו שמות נהגים ששילמו ומכניס אותם ל-driverQueue — בדיוק כמו שהקבלה מוסיפה שם ידנית.

6.3 איך הנתונים נכנסים?

דרך 1 — Webhook (בזמן אמת):

  • Rentix שולח POST ל-/api/webhooks/rentix
  • HAKAFAST מפרסר את ההזמנה (שם, טלפון, רמה) ומוסיף לתור

דרך 2 — סנכרון ידני / מתוזמן:

  • POST ל-/api/webhooks/rentix/sync
  • HAKAFAST מושך הזמנות ששולמו מה-API של Rentix (24 שעות אחרונות)

6.4 שליחת תוצאות חזרה ל-Rentix (אופציונלי)

בסיום מקצה, HAKAFAST יכול לשלוח תוצאות (מיקום, זמן הקפה, מספר הקפות) ל-URL שמוגדר ב-resultsWebhookUrl. זה שימושי אם באתר Rentix רוצים להציג תוצאות ללקוח — אבל לא חובה לתפעול המסלול.

6.5 הגדרות Rentix (install.json / משתני סביבה)

שדהתיאור
publicKey / secretKeyמפתחות API מ-Rentix — לסנכרון הזמנות

8. זרימת עבודה יומית — צעד אחר צעד

8.1 מסלול בלי Rentix (הגיוני לרוב המסלולים)

  1. לקוח מגיע → משלם בקופה (מערכת קופה נפרדת / מזומן / POS)
  2. פקיד קבלה פותח Reception → מוסיף שם + טלפון + רמה לתור
  3. מנהל מקצה ב-Admin רואה את התור → לוחץ "שבץ קארטים"
  4. נהגים עוברים לפיטס → יוצאים למסלול (TranX / לחיצה על טרנספונדר)
  5. הקפות נרשמות אוטומטית → Live Timing מציג
  6. סיום מקצה → ייצוא תוצאות → מקצה הבא

8.2 מסלול עם Rentix

  1. לקוח הזמין ושילם באתר Rentix
  2. Rentix שולח webhook → השם נכנס אוטומטית ל-driverQueue
  3. לקוח walk-in שמשלם בקופה — עדיין נרשם ב-Reception (או Admin)
  4. מנהל מקצה שובץ כרגיל — לא משנה מאיפה הגיע הנהג
  5. אופציונלי: תוצאות חוזרות ל-Rentix לתצוגה באתר

9. מתי כן צריך Rentix ומתי לא

9.1 לא צריך Rentix אם:

  • כל התשלומים בקופה / POS / מזומן
  • אין אתר הזמנות Rentix
  • רוצים רק HAKAFAST + Reception + TranX
  • מסלול חדש שעדיין לא על Rentix

9.2 כן שווה לחבר Rentix אם:

  • כבר מוכרים סessions דרך Rentix ורוצים שהשמות יגיעו אוטומטית לתור
  • רוצים להימנע מהקלדה כפולה (הזמנה באתר + רישום ידני ב-HAKAFAST)
  • רוצים להחזיר תוצאות מקצה לאתר הלקוח
המלצה: התחילו ב-Reception + Admin בלבד. הוסיפו Rentix רק כשיש הזמנות אונליין פעילות ואתם רוצים אוטומציה — לא כי HAKAFAST "דורש" את זה.

10. התקנה במסלול (ישראל) — סקירה

10.1 חומרה ותוכנה

  1. מחשב Windows במסלול (KIOSK) — מריץ HAKAFAST כ-service
  2. TranX 160 (AMB) — מקבל אותות מלולאות מגנטיות
  3. טרנספונדרים בכל קארט — מזהה קארט ביציאה ובסיום הקפה
  4. לולאות — לפחות: pit-out (יציאה), finish (סיום הקפה); אופציונלי: pit-in
  5. רשת סגורה — המחשב, TranX, מסכי Admin/Live/Reception באותה LAN

10.2 התקנת HAKAFAST

  1. הרצת installer (MSI) או npm run build && npm start לפיתוח
  2. Setup Wizard — שם מסלול, workspace, מספר קארטים (אופציונלי בהתחלה)
  3. הגדרת שעות פתיחה / מחיר / משך סsession ב-Track Planner (Admin)
  4. פתיחת Admin: /admin/{track-slug}
  5. פתיחת Reception על טאבלט: /reception/{track-slug}
  6. פתיחת Live Timing על TV: /live-timing/{track-slug}

13. שאלות נפוצות

למה יש גם Reception וגם תור ב-Admin?

אותו תור — שני ממשקים. Reception פשוט לפקיד שלא צריך לראות פיטס ומקצים. Admin מלא למנהל המקצה.

האם "הזמנות" ב-Rentix זה אותו דבר כמו "תור" ב-HAKAFAST?

לא. הזמנה ב-Rentix = עסקת מכירה (מי שילם, מתי, כמה). תור ב-HAKAFAST = מי ממתין physically למקצה הבא. חיבור Rentix רק מעתיק שם מההזמנה לתור.

מה קורה אם אין Rentix בכלל?

הכל עובד. Reception + Admin + TranX = מסלול מלא. Rentix הוא תוסף לאינטגרציה עם אתר קיים.

האם Reception מחליף קופה?

לא. Reception לא גובה כסף. תשלום נשאר בקופה / POS / Rentix.

איך יודעים מאיזה מקור הגיע נהג?

כל נהג בתור יכול לכלול שדה source: ידני, reception, או rentix (במקרה של webhook מ-Rentix).
